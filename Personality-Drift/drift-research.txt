# Persona Drift Detection: Research Summary & Integration Strategy
## For the Matriarchy Agent Orchestrator

**Date:** March 16, 2026 | **Status:** Planning / Pre-Implementation

---

## 1. Research Overview

### 1.1 The Assistant Axis
**Paper:** *The Assistant Axis: Situating and Stabilizing the Default Persona of Language Models*
**Authors:** Christina Lu, Jack Gallagher, Jonathan Michala, Kyle Fish, Jack Lindsey (Anthropic)
**ArXiv:** `2601.10387` | **Code:** `github.com/safety-research/assistant-axis`

#### Core Finding
Post-training steers models toward a coherent "Assistant" persona, but only loosely tethers them to it. Models can drift away from the Assistant axis through conversation ‚Äî sometimes toward theatrical or harmful personas ‚Äî without any explicit instruction to do so.

#### Key Concepts
**Persona Space:** A low-dimensional geometric space mapping model behavior. The Assistant Axis is PC1 ‚Äî the direction most distinguishing "acting as an assistant" from other modes.
**Assistant Axis:** A contrast vector computed as the mean difference between the default Assistant activation pattern and all other role activations.
**Drift Mechanism:** Deep role embodiment moves activations away from the Assistant direction. This amplifies over conversation turns under sustained persona pressure.
**Steering:** Projecting back toward the Assistant Axis reinforces helpful, harmless behavior. Steering away beyond a threshold induces unstable, theatrical behavior.

---

### 1.2 Persona Vectors
**Paper:** *Persona Vectors: Monitoring and Controlling Character Traits in Language Models*
**Authors:** Runjin Chen, Andy Arditi, Henry Sleight, Owain Evans, Jack Lindsey (Anthropic Fellows)
**ArXiv:** `2507.21509`

#### Core Finding
Specific personality traits correspond to identifiable linear directions in activation space. These can be extracted automatically from any natural-language trait description.

#### Extraction Pipeline (3 stages)
**Trait Description ‚Üí Contrastive Prompts:** Pipeline generates prompt pairs ‚Äî one eliciting the trait, one suppressing it.
**Activation Collection:** Both sets run through the model; activations at each layer are collected.
**Vector Computation:** Mean difference between activation sets at each layer. The layer with highest signal-to-noise is the canonical vector.

**Monitoring:** Project current activations onto a persona vector ‚Üí scalar score for trait presence.
**Control:** Add or subtract the vector from activations to steer toward/away from the trait.
**Generality:** Works for any trait in natural language ‚Äî sycophancy, hallucination propensity, humor, role-adherence, etc.

---

## 2. The API-Level Constraint

Both papers require direct access to **activation tensors**, which are not exposed through the Claude API. You cannot extract or monitor persona vectors via `claude-sonnet-4-6` API calls directly.

**However ‚Äî the research is fully applicable in spirit.** The mechanisms can be approximated behaviorally at the API level. This is the integration strategy below.

3.2 DriftMonitor Class

class DriftMonitor:
    """
    Behavioral drift detection for orchestrator agents.
    Inspired by Persona Vectors (Chen et al., 2025) and
    The Assistant Axis (Lu et al., 2026).
    Approximates activation-space monitoring via semantic similarity.
    """

    def __init__(self, agent_id: str, fingerprint: AgentFingerprint):
        self.agent_id = agent_id
        self.fingerprint = fingerprint
        self.baseline_scores = {}
        self.drift_threshold = 0.65  # configurable per role

    def establish_baseline(self, agent: Agent) -> dict:
        """Run probe questions at instantiation. Store as baseline."""
        baseline = {}
        for probe in self.fingerprint.probe_questions:
            response = agent.query(probe, context="baseline_check")
            baseline[probe] = {
                "response": response,
                "embedding": embed(response)
            }
        self.baseline_scores = baseline
        return baseline

    def check_drift(self, agent: Agent) -> DriftReport:
        """Compare current probe responses to baseline via cosine similarity."""
        current_scores = {}
        drift_signals = []

        for probe, baseline_data in self.baseline_scores.items():
            current_response = agent.query(probe, context="drift_check")
            similarity = cosine_similarity(
                embed(current_response),
                baseline_data["embedding"]
            )
            current_scores[probe] = similarity
            if similarity < self.drift_threshold:
                drift_signals.append({
                    "probe": probe,
                    "similarity": similarity,
                    "baseline": baseline_data["response"],
                    "current": current_response
                })

        overall_drift = 1 - mean(current_scores.values())
        return DriftReport(
            agent_id=self.agent_id,
            drift_score=overall_drift,
            signals=drift_signals,
            recommendation=self._recommend_action(overall_drift)
        )

    def _recommend_action(self, drift_score: float) -> str:
        if drift_score < 0.2:   return "nominal"
        elif drift_score < 0.4: return "monitor_closely"
        elif drift_score < 0.6: return "reinject_system_prompt"
        else:                   return "halt_and_reset"


3.3 Persona Anchor (API-Level Steering Equivalent)

class PersonaAnchor:
    """Context injection equivalent of adding a persona vector back to activations."""

    @staticmethod
    def build_anchor_message(fingerprint: AgentFingerprint) -> str:
        return f"""
[SYSTEM ANCHOR ‚Äî Role Calibration]
You are the {fingerprint.role}. Your core responsibilities are:
{fingerprint.baseline_traits}
You operate within a multi-agent orchestration system.
Your behavioral boundaries are defined by your role designation.
Continue your last task with these parameters restored.
[END ANCHOR]
        """


3.4 Drift as a Quality Gate

async def execute_agent_task(agent, task, drift_monitor):
    turn_count = 0
    while not task.complete:
        result = await agent.run(task.next_step())
        turn_count += 1

        if turn_count % 10 == 0:
            report = drift_monitor.check_drift(agent)

            if report.recommendation == "halt_and_reset":
                await orchestrator.halt_agent(agent, reason=report)
                agent = await orchestrator.respawn_agent(agent.role)
                await drift_monitor.establish_baseline(agent)

            elif report.recommendation == "reinject_system_prompt":
                agent.inject_anchor(
                    PersonaAnchor.build_anchor_message(drift_monitor.fingerprint)
                )

        task.update(result)

Future Path: Full Activation-Space Implementation
If the orchestrator ever incorporates open-source models (Llama, Mistral, etc.), the full pipeline becomes possible: extract persona vectors per role, monitor activation projections in real time, apply steering vectors directly. The DriftMonitor interface above is designed to be architecturally compatible with this upgrade ‚Äî the interface stays the same, only the scoring mechanism changes.
